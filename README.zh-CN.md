# AI Coding Engineering Paradigm

[首页](./README.md) | [English](./README.en.md) | 简体中文

这个仓库发布一套带版本管理的源模板包，用于生成仓库级 AI 指令文件；当目标仓库没有更强信号时，默认回退文件名为 `AGENTS.md`。

两个根目录下的 v9 协议文档依然以英文版作为标准维护稿，同时仓库也发布结构对齐的简体中文 companion 文档，方便偏好中文的读者直接使用。

它适用于那些需要 AI 跨越多个会话、贡献者与交接过程仍保持连贯的仓库。在这些场景里，AI 需要把可持续读取的工作记忆留在仓库中。

这是一套文档包和工作模式，不是软件框架、起始代码库，也不是编排平台。

## 为什么需要这个仓库

AI 在短对话里可以很高效，但一旦进入长周期工作，表现往往会明显下降。

常见失败模式通常很稳定：

- 忘记已经做过的决策、假设和背景信息
- 重复已经完成的调研和发现工作
- 丢失当前状态和真正的下一步
- 让跨会话或跨协作者的交接变得更困难
- 留下难以审计和维护的工作痕迹

对于长期任务，问题通常不只是模型能力本身。更常见的根源是：过多项目记忆被困在临时聊天上下文里。

这套模板包的作用，就是帮助 agent 为目标仓库生成一个规范指令文件，以及目标仓库真正需要的最小支持记忆结构。

## 核心思想

把仓库文档当作外置工作记忆。

不要期待模型记住所有历史对话，而是要求它把重要状态写进人类和未来 AI 会话都能读取的文件中：

- 稳定规则
- 需要时才启用的当前状态
- 非平凡任务的计划
- 以后还会重要的持久决策
- 可复用的起始模板

仓库因此成为项目记忆的承载面。上下文可以跨越会话重置、长时间暂停、模型升级，以及人类或 AI 之间的交接而继续保留。

## 模板产出什么

生成后的目标仓库，只应保留那些真正被其工作流证明有价值的层。

- 一个规范的 AI 指令文件
- 只有在能防止真实上下文丢失时才启用的状态、计划和决策层
- 仅在更长、更模糊、评审更重或多 agent 协作任务里才启用的 `spec` / requirements 层
- 贴合目标仓库实际情况的路径选择，而不是机械复刻本仓库

关键点是把稳定规则和易变工作状态分开，而不是强迫所有仓库采用同一套固定目录树。

## 适用场景

当工作是长周期、迭代式，或者很容易失去上下文时，这个模板就有价值。

典型场景包括：

- 跨越多个会话或贡献者的软件开发
- 模型训练与评估循环
- 研究和实验密集型仓库
- 频繁上下文切换的项目
- 依赖清晰交接的人机协作
- 需要可追溯性和可维护决策历史的工作

## 仓库内容

这个仓库把最新推荐模板源包保留在仓库根目录和 `.ai/templates/` 下。

- 最新公开模板资产：`AGENTS.md`、`README.md`、`README.en.md`、`README.zh-CN.md`、`repo_ai_instruction_generation_protocol_v9.md`、`repo_ai_instruction_generation_protocol_v9.zh-CN.md`、`repo_ai_instruction_generation_protocol_user_guide_v9.md`、`repo_ai_instruction_generation_protocol_user_guide_v9.zh-CN.md` 与 `.ai/templates/`
- [VERSIONS.md](VERSIONS.md)：当前版本与历史快照的统一索引
- [versions/](versions/)：历史公开快照目录，例如 `versions/2026-04-02/`
- `.ai/tracking.md`、`.ai/plans/` 和 `.ai/decisions/` 这类仅限本地的维护文件可以用于维护本仓库的工作副本，但它们不属于对外发布的模板包，也不应成为远端快照的依赖。

维护这个仓库本身时，也可以在本地使用同样的文档驱动方式；但这些维护文件会被有意排除在公开源包和远端快照之外，不应被默认当成目标仓库要复制的模板资产。

## 版本如何工作

- `repo_ai_instruction_generation_protocol_v9.md` 这类文件名里的 `v9` 表示协议代际，不表示模板包发布日期版本。
- 模板包版本使用日期格式，例如 `2026-04-02`。
- 同一天内的后续修订使用 `2026-04-02-r2`、`2026-04-02-r3` 这类后缀。
- 仓库根目录和 `.ai/templates/` 始终表示最新推荐版本；历史公开快照放在 `versions/<release-id>/`。
- 模板包发布的 Git tag 应使用 `template-<release-id>`。现有的 `V9` tag 继续保留为历史协议里程碑。

## 如何使用这个模板

1. 先看 [VERSIONS.md](VERSIONS.md)。如果要最新版本，就使用仓库根目录和 `.ai/templates/`；如果要历史版本，就选择 `versions/<release-id>/` 里的快照。
2. 为目标仓库生成或刷新一个规范 AI 指令文件。只有在目标仓库没有更强信号时，才把 `AGENTS.md` 当成默认回退文件名。
3. 只为目标仓库启用真正有必要的支持记忆层。
4. 优先复用目标仓库已经有效的路径和组织方式，不要机械照搬本仓库。
5. 如果你在维护这个源仓库的本地工作副本，可把 `.ai/tracking.md`、`.ai/plans/` 和 `.ai/decisions/` 视为源仓库自身的可选本地维护文件，不要默认把它们当成公开模板资产复制出去。
